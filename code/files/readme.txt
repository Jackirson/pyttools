Алгоритм ранжирования технологий по их инвестиционной привлекательности выполнен в виде набора скриптов пакета прикладных программ Matlab и требует для своей работы наличия установленного пакета Matlab версии не ниже R2013b.

Для запуска алгоритма ранжирования необходимо выполнить следующие действия:

1. Поместить файлы main_sort.m, vectorS1.m, load_p.m, save_p.m, face.m, dichotomy.m, sum_function.m и gui.m в одну директорию, на которую пользователь имеет права чтения и записи.

2. Данные по оценке показателей инвестиционной привлекательности технологий, предоставленные экспертами, поместить в файлы expert1.txt, expert2.txt и т. д. Каждый файл содержит информацию, предоставленную одним экспертом, и имеет формат аналогичный следующему:

В первой строке файла после текста "I[Techn.count]=" указывается количество оцениваемых технологий. Данные о каждой технологии помещаются в файл в виде блока из 16-ти строк (нечёткие оценки по 16-ти показателям, характеризующим инвестиционную привлекательность, берутся из таблиц в форме оценки технологии), которому предшествует текст "Technology #XXX", где XXX - трёхзначный номер технологии. Технологии нумеруются, начиная с единицы, и входят в данный файл последовательно (сначала 1-ая, затем 2-ая, и т. д.).

3. Названия файлов, содержащих нечёткие экспертные оценки, следует поместить в начало файла main_sort.m в список filename_list, например:

    filename_list = {'expert1.txt', 'expert2.txt', 'expert3.txt', 'expert4.txt'};

4. В консоли Matlab-а перейти в директорию, содержащую указанные в пунктах 1 и 2 файлы, с помощью команды "cd имя_директории", после чего выполнить команду "main_sort". Здесь имя_директории следует заменить на путь к директории, в которую требуется осуществить переход.

Краткое описание файлов:

sum_function.m
Реализация числовой функции, выражающей зависимость инвестиционной привлекательности технологии от 16-ти показателей, её определяющих.

face.m
Выполняет вычисление возможностей потерь, соответствующих включению каждой из оцениваемых технологий в список наиболее привлекательных.

dichotomy.m
Вспомогательная функция для алгоритма face.m.

load_p.m
Осуществляет загрузку файлов, содержащих нечёткие экспертные оценки.

save_p.m
Осуществляет сохранение файлов, содержащих нечёткие экспертные оценки.

vectorS1.m
Алгоритм формирования коллективной экспертизы на основе данных, полученных от группы экспертов.

main_sort.m
Алгоритм ранжирования технологий.

gui.m
Графический интерфейс, позволяющий формировать файлы с нечёткими экспертными оценками.
